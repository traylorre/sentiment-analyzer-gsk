# Implementation Plan: Chaos Security Hardening

**Branch**: `G-chaos-security-hardening` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)

## Summary

Harden chaos testing API against unauthenticated access, enforce experiment duration via auto-restore, add rate limiting and concurrent experiment prevention, and emit IAM change alerts. All changes are application-level (Python) plus minimal Terraform for the CloudWatch alarm and EventBridge Scheduler IAM role.

## Key Design Decisions

1. **EventBridge Scheduler (not Rules)**: Scheduler supports `at()` one-time triggers with `ActionAfterCompletion=DELETE` for self-cleanup. Rules only support cron/rate.

2. **DynamoDB conditional writes for rate limiting and locking**: Atomic, strongly consistent, no new infrastructure. Uses synthetic `experiment_id` values (`RATELIMIT#chaos#{user_id}` and `CHAOSLOCK#{scenario_type}`) in the existing chaos table (which uses `experiment_id` as hash key, `created_at` as range key).

3. **Reuse Dashboard Lambda for auto-restore**: The Lambda already has all required IAM permissions. Auto-restore is handled in `lambda_handler()` BEFORE Powertools routing via raw event check (`event.get("action") == "chaos-auto-restore"`), because EventBridge Scheduler invokes Lambda directly — not via HTTP.

4. **404 before auth (defense in depth)**: Environment check is the FIRST thing in each chaos handler — before even extracting the user_id. Attackers learn nothing about whether chaos endpoints exist in prod/preprod.

5. **Schedule before status update**: Auto-restore is scheduled BEFORE experiment status changes to `running`. If scheduling fails, the experiment is immediately restored and marked `failed`. This prevents the "degraded with no auto-restore" state.

## Risk Mitigation

- **Auto-restore reliability**: EventBridge Scheduler has built-in retries. If the Lambda target fails, Scheduler retries with exponential backoff.
- **Rate limit bypass via DynamoDB failure**: Fail-closed — if conditional write fails for any reason other than `ConditionalCheckFailedException`, reject with 503.
- **Lock leak**: Lock items have TTL (10 minutes). If `stop_experiment()` fails to release the lock, it auto-expires via DynamoDB TTL.
- **Feature 1249 dependency**: PR #798 is in auto-merge queue. If it hasn't merged, this feature includes a standalone `_is_dev_environment()` fallback.
